IPV4 WWAN

As the page loads the applied configuration is displayed by default, by using dropdown menu and text box you can change IPV4 Configuration

IPV4 Support:

This dropdown enables/ disables IPv4 functionality. The IPv4 call is established using the stored network policy if auto connect is enabled.

Backhaul:

WWAN backhaul bring up using the network policy in the xml file. Connect to backhaul request will trigger connect to the WWAN Network .

WWAN Policy:

You can view and update the WWAN Policy information

Tech preference:

A single connection is attempted using the following specified technology preferences

3GPP profile id:

Shows CDMA profile ID for IPv4. You can edit it using text box.

3GPP2 profile id:

Shows UMTS profile ID for IPv4. You can edit it using text box.

To save and cancel changes

Click UPDATE to save the new setting. Click REFRESH to discard any unsaved changes.

IPV4 WWAN Statistics

You can display and reset WWAN statistics along with WWAN IP configuration for the mobile AP as Bytes received on WWAN. Bytes transmitted on WWAN. Packets received on WWAN. Packets transmitted on WWAN. Packets dropped on Rx WWAN. Packets dropped on Tx WWAN. IPv4 public IP IPv4 primary DNS address IPv4 Secondary DNS Address

STATION mode connection status

In AP-STA mode, our MTP works as an AP and as a station to connect to the external hotspot at the same time. When external hotspot is available, backhaul connection is switched to WLAN backhaul and when external hotspot is not available backhaul connection is switched back to WWAN dynamically. Also data bit rate is disaplyed and can be refreshed by clicking on CURRENT DATA RATE button.
